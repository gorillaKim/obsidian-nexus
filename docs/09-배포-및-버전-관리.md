# 배포 및 버전 관리

## 버전 관리

버전은 **두 곳**에서 관리하며, 반드시 동일하게 유지해야 한다:

| 파일 | 필드 |
|------|------|
| `Cargo.toml` (워크스페이스) | `workspace.package.version` |
| `apps/desktop/src-tauri/tauri.conf.json` | `version` |

워크스페이스 버전은 `version.workspace = true`를 통해 모든 crate에 자동 전파된다.

## 릴리스 프로세스

```bash
# 1. 두 파일의 버전을 동일하게 올린다
#    Cargo.toml: version = "0.2.0" → "0.3.0"
#    tauri.conf.json: "version": "0.2.0" → "0.3.0"

# 2. 커밋
git add Cargo.toml apps/desktop/src-tauri/tauri.conf.json
git commit -m "chore: bump version to 0.3.0"

# 3. 태그 생성 + 푸시
git tag v0.3.0
git push && git push origin v0.3.0
```

태그 푸시 시 `.github/workflows/release.yml`이 자동 트리거된다.

## 자동 업데이트 아키텍처

```
Desktop: tauri_plugin_updater (빌트인) → @tauri-apps/plugin-updater JS API
CLI:     nexus update → GitHub Release API + 바이너리 교체
```

### Desktop 업데이트

- **Tauri Updater 플러그인** 사용 (커스텀 Rust 코드 없음)
- `UpdateChecker` 컴포넌트 (`apps/desktop/src/components/UpdateChecker.tsx`)
  - 헤더: 새 버전 감지 시 업데이트 배지 표시
  - 설정 탭: 현재 버전 + "업데이트 확인" 버튼 + 진행률
- 앱 시작 시 백그라운드로 `latest.json` endpoint 체크
- 서명 검증: `tauri.conf.json`의 `pubkey`로 다운로드된 번들 검증
- 업데이트 후 자동 재시작 (`@tauri-apps/plugin-process`의 `relaunch()`)

### CLI 업데이트

```bash
nexus update          # 확인 + 설치
nexus update --check  # 확인만
nexus update --force  # 24시간 캐시 무시
```

- GitHub Release API로 최신 버전 확인 (`tag_name` vs `CARGO_PKG_VERSION`)
- `std::env::consts::ARCH`로 아키텍처 자동 감지 (aarch64/x86_64)
- **SHA256 체크섬 필수 검증** (체크섬 파일 없으면 설치 거부)
- Atomic 바이너리 교체:
  1. `.new` 파일로 다운로드 + 검증
  2. 기존 바이너리를 `.bak`으로 백업
  3. `.new` → 최종 경로로 rename
  4. 실패 시 `.bak`에서 자동 복원
- 24시간 체크 캐시 (`~/.nexus/update-check`)
- Desktop 앱 번들(`/Applications/`)은 CLI로 교체하지 않음 — Tauri updater가 담당

## CI/CD 구조

### GitHub Actions 워크플로우

`.github/workflows/release.yml` — `v*` 태그 푸시 시 트리거

#### Desktop Job (`desktop`)

1. 두 아키텍처의 sidecar 빌드 (nexus, nexus-mcp-server)
2. `cargo tauri build --target universal-apple-darwin`
3. `latest.json` 생성 (서명 포함)
4. GitHub Release에 업로드:
   - `Obsidian Nexus_X.Y.Z_universal.dmg` (신규 설치용)
   - `Obsidian Nexus.app.tar.gz` (업데이트 번들)
   - `Obsidian Nexus.app.tar.gz.sig` (서명)
   - `latest.json` (updater endpoint)

#### CLI Job (`cli`, needs: desktop)

Matrix: `macos-14` (aarch64) + `macos-13` (x86_64)

1. `cargo build --release -p nexus-cli -p nexus-mcp-server`
2. `tar.gz` 아카이브 + SHA256 체크섬 생성
3. GitHub Release에 업로드:
   - `nexus-cli-darwin-{arch}.tar.gz`
   - `nexus-cli-darwin-{arch}.tar.gz.sha256`

### GitHub Secrets

| Secret | 용도 |
|--------|------|
| `TAURI_SIGNING_PRIVATE_KEY` | Tauri 번들 서명 개인키 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 개인키 비밀번호 (없으면 빈 값) |

## 서명 키 관리

### 키 생성

```bash
cargo tauri signer generate -w ~/.tauri/obsidian-nexus.key
```

- 개인키: `~/.tauri/obsidian-nexus.key` (GitHub Secrets에 등록)
- 공개키: `~/.tauri/obsidian-nexus.key.pub` (`tauri.conf.json`의 `pubkey`에 설정)

### 주의사항

- **개인키 분실 시 업데이트 서명 불가** → 백업 필수
- 공개키는 앱에 내장되므로 개인키가 바뀌면 모든 사용자가 재설치해야 함
- CI에서만 서명 가능 (로컬 빌드는 서명 없음)

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| CI signing 실패 | `TAURI_SIGNING_PRIVATE_KEY` Secret 미등록 | GitHub Secrets 확인 |
| `latest.json` 404 | desktop job 실패 | Actions 로그 확인 |
| Desktop 업데이트 감지 안 됨 | `tauri.conf.json` 버전 미변경 | 두 파일 버전 bump 확인 |
| CLI `nexus update` 에러 | release에 CLI asset 없음 | cli job 로그 확인 |
| sidecar 빌드 실패 | `binaries/` 경로에 바이너리 없음 | sidecar 빌드 단계 확인 |
| Gatekeeper 차단 (.dmg) | macOS notarization 없음 | 우클릭 → 열기로 우회 |

## 향후 과제

- **macOS notarization**: Apple Developer 계정 확보 후 CI에 `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID` 추가
- **Linux/Windows 지원**: CI matrix 확장 + CLI asset name에 OS 반영 (이미 코드에 `std::env::consts::OS` 사용)
- **자동 버전 bump**: `cargo-release` 또는 CI에서 자동 처리
