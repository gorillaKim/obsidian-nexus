# 배포 및 버전 관리

## 빠른 릴리스 (TL;DR)

```bash
./scripts/bump-version.sh        # 패치 자동 증가 (0.3.0 → 0.4.0)
./scripts/bump-version.sh 1.0.0  # 특정 버전 지정
```

이 스크립트가 버전 bump → 커밋 → 태그 → 푸시 → CI 트리거를 자동 처리한다.

## 버전 관리

버전은 **세 곳**에서 관리하며, 반드시 동일하게 유지해야 한다:

| 파일 | 필드 | 용도 |
|------|------|------|
| `Cargo.toml` (워크스페이스) | `workspace.package.version` | Rust crate 버전 (모든 crate에 전파) |
| `apps/desktop/src-tauri/tauri.conf.json` | `version` | Tauri 앱 + updater 버전 비교 |
| `apps/desktop/package.json` | `version` | 프론트엔드 표시용 (`__APP_VERSION__`) |

> `bump-version.sh`가 세 파일을 동시에 업데이트한다.

## 릴리스 프로세스

### 자동 (권장)

```bash
./scripts/bump-version.sh
```

### 수동

```bash
# 1. 세 파일의 버전을 동일하게 올린다
# 2. 커밋
git add Cargo.toml apps/desktop/src-tauri/tauri.conf.json apps/desktop/package.json
git commit -m "chore: bump version to X.Y.Z"

# 3. 태그 생성 + 푸시
git tag vX.Y.Z
git push && git push origin vX.Y.Z
```

태그 푸시 시 `.github/workflows/release.yml`이 자동 트리거된다.

## 자동 업데이트 아키텍처

```
Desktop: tauri_plugin_updater (빌트인) → @tauri-apps/plugin-updater JS API
CLI:     nexus update → GitHub Release API + 바이너리 교체
```

### Desktop 업데이트

- **Tauri Updater 플러그인** 사용 (커스텀 Rust 코드 없음)
- `UpdateChecker` 컴포넌트 (`apps/desktop/src/components/UpdateChecker.tsx`)
  - 설정 탭: 현재 버전 + "업데이트 확인" 버튼 + 진행률
- 앱 시작 시 백그라운드로 `latest.json` endpoint 체크
- 서명 검증: `tauri.conf.json`의 `pubkey`로 다운로드된 번들 검증
- 업데이트 후 자동 재시작 (`@tauri-apps/plugin-process`의 `relaunch()`)

#### 필수 설정 (tauri.conf.json)

```json
{
  "bundle": {
    "active": true,
    "createUpdaterArtifacts": true
  },
  "plugins": {
    "updater": {
      "endpoints": ["https://github.com/gorillaKim/obsidian-nexus/releases/latest/download/latest.json"],
      "pubkey": "<공개키>"
    }
  }
}
```

- `bundle.active: true` — 없으면 번들링 자체가 스킵됨
- `bundle.createUpdaterArtifacts: true` — 없으면 `.app.tar.gz` + `.sig` 미생성 (Tauri v2 필수)
- `plugins.updater.active`는 v1 문법이므로 v2에서는 무시됨

### CLI 업데이트

```bash
nexus update          # 확인 + 설치
nexus update --check  # 확인만
nexus update --force  # 24시간 캐시 무시
```

- GitHub Release API로 최신 버전 확인
- `std::env::consts::ARCH`로 아키텍처 자동 감지 (aarch64/x86_64)
- **SHA256 체크섬 필수 검증** (체크섬 파일 없으면 설치 거부)
- Atomic 바이너리 교체 (`.new` → `.bak` 백업 → rename → 실패 시 복원)
- 24시간 체크 캐시 (`~/.nexus/update-check`)
- Desktop 앱 번들은 CLI로 교체하지 않음 — Tauri updater가 담당

## CI/CD 구조

### GitHub Actions 워크플로우

`.github/workflows/release.yml` — `v*` 태그 푸시 시 트리거

#### Desktop Job

1. aarch64 + x86_64 sidecar 빌드 (nexus, nexus-mcp-server)
2. `lipo`로 universal sidecar 생성
3. `cargo tauri build --target universal-apple-darwin`
4. `latest.json` 생성 (서명 포함)
5. GitHub Release에 업로드: `.dmg`, `.app.tar.gz`, `.sig`, `latest.json`

#### CLI Job (needs: desktop)

1. `macos-14`에서 cross-compile (aarch64 + x86_64)
2. 아키텍처별 `tar.gz` + SHA256 체크섬 생성
3. GitHub Release에 업로드

#### 빌드 캐시

- `actions/cache@v4`로 `~/.cargo/registry`, `target/` 캐싱
- Tauri CLI 바이너리 캐싱 (`which cargo-tauri`로 재설치 스킵)

### GitHub Secrets

| Secret | 용도 | 설정 방법 |
|--------|------|-----------|
| `TAURI_SIGNING_PRIVATE_KEY` | Tauri 번들 서명 개인키 | `cat ~/.tauri/obsidian-nexus.key \| pbcopy` → Secret에 붙여넣기 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 개인키 비밀번호 | `--ci`로 생성했으면 빈 값 |

## 서명 키 관리

### 키 생성

```bash
cargo tauri signer generate -w ~/.tauri/obsidian-nexus.key --ci
```

- 개인키: `~/.tauri/obsidian-nexus.key` → GitHub Secrets에 등록
- 공개키: `~/.tauri/obsidian-nexus.key.pub` → `tauri.conf.json`의 `pubkey`에 설정

### 주의사항

- **개인키 분실 시 업데이트 서명 불가** → 백업 필수
- 공개키는 앱에 내장되므로 개인키가 바뀌면 모든 사용자가 재설치해야 함
- `TAURI_SIGNING_PRIVATE_KEY` 환경변수가 있어야 CI에서 `.sig` 파일 생성

## 개발 환경에서 앱 실행

```bash
cargo tauri dev    # 프론트엔드(Vite) + Tauri 앱 hot-reload
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 번들 미생성 (Built만 출력, Bundling 없음) | `bundle.active: true` 누락 | tauri.conf.json에 추가 |
| `.app.tar.gz` 미생성 | `createUpdaterArtifacts: true` 누락 | tauri.conf.json에 추가 |
| `latest.json`의 signature가 빈 값 | `TAURI_SIGNING_PRIVATE_KEY` Secret 미등록 | GitHub Secrets 확인 |
| `latest.json` 404 | 비공개 레포 또는 desktop job 실패 | 레포 공개 전환 또는 Actions 로그 확인 |
| updater check 결과 null | 현재 버전 = latest 버전 (업데이트 없음) | 정상 동작. 버전을 올려서 테스트 |
| updater check 에러 후 "최신 버전" 표시 | signature 빈 값 → 검증 실패 → null 반환 | CI에서 서명 생성 확인 |
| CI `frozen-lockfile` 에러 | lockfile에 새 의존성 미반영 | `pnpm install` 후 lockfile 커밋 |
| sidecar 빌드 실패 (`resource path doesn't exist`) | universal sidecar 미생성 | `lipo -create`로 universal 바이너리 생성 |
| `macos-13` runner 에러 | GitHub Actions에서 지원 중단 | `macos-14`에서 cross-compile |
| `__APP_VERSION__` 이 0.1.0 고정 | `npm_package_version` 환경변수 미주입 | vite.config.ts에서 tauri.conf.json 직접 읽기 |
| Gatekeeper 차단 (.dmg) | macOS notarization 없음 | 우클릭 → 열기로 우회 |

## Release 산출물

| 파일 | 용도 |
|------|------|
| `Obsidian-Nexus.dmg` | 신규 설치용 (universal) |
| `Obsidian.Nexus.app.tar.gz` | Desktop 자동 업데이트 번들 |
| `Obsidian.Nexus.app.tar.gz.sig` | 번들 서명 |
| `latest.json` | Tauri updater endpoint |
| `nexus-cli-darwin-aarch64.tar.gz` | CLI (Apple Silicon) |
| `nexus-cli-darwin-x86_64.tar.gz` | CLI (Intel) |
| `*.sha256` | CLI 체크섬 |

## 향후 과제

- **macOS notarization**: Apple Developer 계정 확보 후 CI에 `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID` 추가
- **Linux/Windows 지원**: CI matrix 확장 + CLI asset name에 OS 반영
